Size, Position & Lists

1. px – Pixels (Fixed Size)

  • Definition: A pixel is a fixed unit representing a single dot on the screen.
  • Use Case: Best for precise control (e.g., borders, spacing, icons).
  • Responsive? ❌ Not responsive. Doesn't scale with screen size

Example:

div {
  width: 200px;  /* Always 200 pixels wide */
}

200px wide

2. % – Percentage of Parent Element

  • Definition: A relative unit. The size is calculated as a percentage of the parent element's size.
  • Use Case: Makes layouts fluid and adaptable.
  • Responsive? ✅ Yes, responds to parent size.

Example:


.parent {
  width: 400px;
}

.child {
  width: 50%;  /* 50% of 400px = 200px */
}

3. em – Relative to Parent Font Size

  • Definition: Relative to the font-size of the parent element.
  • 1em = current font size.
  • Use Case: Good for scaling paddings, margins, or font-size relative to surrounding text.
  • Responsive? ✅ Depends on parent.

Example:


body {
  font-size: 16px;
}

p {
  font-size: 1.5em; /* 1.5 × 16px = 24px */
}

Parent font size (16px)

Child font size (1.5em)

4. rem – Relative to Root Font Size

  • Definition: Relative to the root (html) font-size, not the parent.
  • Use Case: More consistent scaling than em.
  • Responsive? ✅ Yes, based on root size (easy to manage across the site).

Example:


html {
  font-size: 16px;
}

h1 {
  font-size: 2rem; /* 2 × 16px = 32px */
}
Root Element [Html] Font Size 16px.
Child Box font-size: 2rem; 2 × 16px = 32px

5.vw & vh – Viewport Width / Height

  • 1vw = 1% of the viewport width
  • 1vh = 1% of the viewport height
  • Use Case: Responsive designs that adapt to screen size regardless of container or parent.
  • Responsive? ✅ Highly responsive.

Example:


div {
  width: 50vw;  /* 50% of the browser width */
  height: 100vh; /* Full screen height */
}
width: 50vw; 50% of the browser width height: 100vh; Full screen height
